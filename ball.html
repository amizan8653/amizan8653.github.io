ball¶

This is the module that contains the ball class.

class ball.ball(trueBallPos, maxBallSpeed, inPosession=False)¶

The Ball Class

The functionality of the ball class is to keep track of ball’s current position, current direction it’s moving, whether or not a player is in possession of it, and it stores a maximum speed that it can be passed at.

Parameters:
  • trueBallPos (float,float) – a tuple of 2 floats representing the true position of the ball on the field
  • maxBallSpeed (float) – a floating point value indicating the maximum speed a ball can be passed. by default, agents will pass the ball as fast as possible
  • inPosession (boolean) – indicates if a keeper or taker currently has possession of ball.
updateCoordinate(newCoord)¶

update ball coordinate

This function will update the true ball’s true position on the field.

Parameters:newCoord (float, float) – a tuple representing the true ball coordinates
updateDirection(newDirectionVector)¶

update the direction component of ball velocity

This function is called upon by the agent class to attempt an update to update the direction that the ball is going. The direction is stored as a unit vector.

Parameters:newDirectionVector (float, float) – a tuple representing the direction that the agent is trying to change the ball to. this vector does not need to be in the form of a unit vector. The direction will only be set if the ball is NOT in posession. If the ball is in possession, the the direction will remain (0.0,0.0).
Returns:no return
updatePosession(possession)¶

update whether or not someone has posession of the ball

Parameters:posession – if true, then the ball is in possession. If false, then the ball is NOT in possession. If ball is in posession, also set the ball direction value to (0.0, 0.0)
Returns:no return
